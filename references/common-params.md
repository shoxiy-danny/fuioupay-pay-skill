# 富友支付 - 公共请求参数

所有富友支付接口的请求报文，均需包含以下公共参数：

## 必填公共参数

| 参数名 | 必填 | 类型 | 长度 | 说明 |
|--------|------|------|------|------|
| version | 是 | String | 8 | 固定值 `1.0` |
| ins_cd | 是 | String | 13 | 机构号，由富友分配 |
| mchnt_cd | 是 | String | 15 | 商户号，由富友分配 |
| term_id | 是 | String | 8 | 终端号，无真实终端统一填 `88888888` |
| random_str | 是 | String | 32 | 随机字符串，建议使用UUID或随机32位字符串 |
| sign | 是 | String | 512 | RSA签名值，Base64编码 |

## 公共参数说明

### version
固定填写 `1.0`，标识接口版本。

### ins_cd（机构号）
由富友分配的机构唯一标识，长度13位。

### mchnt_cd（商户号）
由富友分配的商户唯一标识，长度15位。

### term_id（终端号）
商户下的终端标识，长度8位。无真实终端时统一填写 `88888888`。

### random_str（随机字符串）
用于防止重放攻击，建议每次请求生成新的32位随机字符串。

### sign（签名）
对请求参数进行RSA签名后的值，详见 [签名详解](../guide/signature.md)。

## 订单相关公共参数

部分接口还包含以下订单公共参数：

| 参数名 | 必填 | 类型 | 长度 | 说明 |
|--------|------|------|------|------|
| mchnt_order_no | 是 | String | 30 | 商户订单号，5-30字符，字母数字下划线，全局唯一 |
| order_amt | 是 | Number | 9 | 订单金额，**单位：分** |
| txn_begin_ts | 是 | String | 14 | 交易开始时间，格式 `yyyyMMddHHmmss` |

## 公共响应参数

所有接口响应均包含以下公共字段：

| 参数名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| result_code | 是 | String | 错误码，`000000` 表示成功 |
| result_msg | 是 | String | 错误描述信息 |
| ins_cd | 是 | String | 机构号（原样返回）|
| mchnt_cd | 是 | String | 商户号（原样返回）|
| random_str | 是 | String | 随机字符串（原样返回）|
| sign | 是 | String | 响应签名（需验签）|

## 注意事项

1. **金额单位**：所有金额类参数单位均为**分**，非元
2. **字符编码**：所有参数均使用 **GBK** 编码
3. **签名排除**：参与签名计算时，`sign` 字段和 `reserved` 开头的字段不参与
4. **空字段**：报文中空字段格式为 `<field></field>`，不能省略
