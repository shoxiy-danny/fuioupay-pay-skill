# 3.15 交易手续费推送接口

## 请求地址

合作方需提供回调地址配置，发送邮件到：`techsupport@fuioupay.com`

邮件主题：配置交易手续费推送回调地址

邮件内容：机构号/商户号，机构名称/商户名称，回调地址，说明是配置交易手续费推送回调地址。

## 场景说明

富友通过此接口向商户推送交易手续费信息。

- 商户维度配置：使用富友默认密钥验签
- 代理维度配置（一代和二代）：如未配置密钥，使用默认公钥验签

**响应要求：** 接收回调成功处理成功后返回字符串 `1`，富友停止回调。最多回调5次，每次间隔15秒。

## 支持的交易类型

推送交易类型为：TX09, TX15, TX27, TX37, TX42, TX23, TX91, TX93, CX91, CX93, TX18, TX02, TX03, CX02, TX01, CX01, TX10, TX68, TX70, PY70, PY71, PY68, PY69, TX66, CX68, CX70, CX69, CX71, PX52, PX57, PX61, PX63, PX64, PX11, PX15

**特殊说明：**
- 刷卡类交易的撤销和退款交易（TX03, CX02, CX01, CX68, CX70, CX69, CX71, CX00）：`orig_mchnt_order_no`字段展示"原交易日期（yyyyMMdd）+原交易本地跟踪号"
- 结算类交易：`mchnt_order_no`为空

**请求报文：**

```xml
    <orig_mchnt_order_no></orig_mchnt_order_no>
    <reserved_fas_settle_dt>20201202</reserved_fas_settle_dt>
    <reserved_receivable_amt>10</reserved_receivable_amt>
    <reserved_paidin_amt>10</reserved_paidin_amt>

    <txn_amt>1</txn_amt>
    <txn_fin_ts>20201202173828</txn_fin_ts>

```

**签名原文：**

```
channel_order_no=4200000742202012028596001565&fee_amt=0&ins_cd=08K0023029&kbps_trace_no=120497549430&mchnt_cd=0004510F3217505&mchnt_order_no=E20201202171223382229376/TX003542&orig_mchnt_order_no=&txn_amt=1&txn_fin_ts=20201202173828
```

## 验签说明

1. 接收回调后，对请求body中的数据进行URLDecode获得XML报文
2. 使用富友提供的公钥进行RSA验签
3. 验签时需排除sign字段，签名原文按字典序拼接key=value&格式
4. 验签通过后返回字符串 `1`

## 注意事项

1. 手续费金额 `fee_amt` 已废弃，建议使用 `reserved_paidin_amt` 和 `reserved_receivable_amt` 计算
2. 减免手续费 `reserved_discount_amt` 已废弃，建议用 `reserved_paidin_amt - reserved_receivable_amt` 计算
3. 退款交易手续费为负数
4. 结算类交易的 `mchnt_order_no` 字段为空
5. 刷卡类退款的 `orig_mchnt_order_no` 包含原交易日期和跟踪号，需自行解析关联
