# 3.5 支付结果通知（回调）

## 请求地址

回调地址在请求报文的 `notify_url` 字段中指定，测试和生产环境都是如此。

## 场景说明

下单（主扫）交易的结果是以**异步**的形式进行回调的。富友在接收到支付宝等支付通道的回调结果以后再回调商户。

**重要**: 不保证通知最终一定能成功，在订单状态不明或者没有收到支付结果通知的情况下，建议商户主动调用【3.4 订单查询】确认订单状态。

**只有主扫统一下单、公众号/服务窗统一下单会通过此接口发异步通知，条码支付没有异步通知。**

## 回调格式

```
POST
Content-Type: application/x-www-form-urlencoded
req={encode之后的xml报文}
```

收到回调后直接在request里面获取`req`参数的值，然后decode一下就得到xml格式报文。

## 回调通知请求报文

| 序号 | 参数名 | 必填 | 类型 | 说明 |
|------|--------|------|------|------|
| 1 | result_code | 是 | String | 错误代码，000000成功 |
| 2 | result_msg | 是 | String | 错误描述 |
| 3 | ins_cd | 是 | String | 机构号 |
| 4 | mchnt_cd | 是 | String | 商户号 |
| 5 | term_id | 否 | String | 终端号 |
| 6 | random_str | 是 | String | 随机字符串 |
| 7 | sign | 是 | String | 签名 |
| 8 | user_id | 否 | String | 用户在商户的ID |
| 9 | order_amt | 是 | Number | 订单金额，单位：分 |
| 10 | settle_order_amt | 是 | Number | 应结订单金额 |
| 11 | curr_type | 否 | String | 货币类型，默认CNY |
| 12 | transaction_id | 是 | String | 渠道订单号 |
| 13 | mchnt_order_no | 是 | String | 商户订单号 |
| 14 | order_type | 否 | String | 订单类型 |
| 15 | txn_fin_ts | 是 | String | 支付完成时间yyyyMMddHHmmss |
| 16 | reserved_fy_settle_dt | 是 | String | 富友交易日期 |
| 17 | reserved_coupon_fee | 否 | String | 优惠金额（分）|
| 18 | reserved_buyer_logon_id | 否 | String | 买家渠道登录账号 |
| 19 | reserved_fy_trace_no | 否 | String | 富友系统内部追踪号 |

| 21 | reserved_is_credit | 否 | String | 1-信用卡/花呗 |
| 22 | reserved_bank_type | 否 | String | 付款方式 |

## 回调响应

商户接收回调成功处理成功后，直接往流里写入：**数字1**

**最多回调6次，每次间隔30秒。**

## 代码示例

```javascript
app.post('/callback', async (req, res) => {
  try {
    // 获取req参数
    const encodedXml = req.body.req;

    // URL解码
    const xmlData = doubleDecodeAndGbk(encodedXml);

    // 解析XML
    const data = await parseXml(xmlData);
    console.log('回调数据:', data);

    // 验签（注意：sign可能含空格，需要过滤）
    const cleanSign = data.sign.replace(/\s/g, '');
    const verifyContent = buildSignContent(data);
    const isValid = verifyWithPublicKey(verifyContent, cleanSign, publicKey);

    if (!isValid) {
      console.error('验签失败');
      return res.send('FAIL');
    }

    // 处理业务
    if (data.result_code === '000000') {
      const orderNo = data.mchnt_order_no;
      const amount = data.order_amt;

      // 更新订单状态
      await updateOrder(orderNo, 'success', amount);
    }

    // 返回成功
    res.send('1');
  } catch (e) {
    console.error('回调处理异常:', e);
    res.send('FAIL');
  }
});
```

## 注意事项

1. **幂等处理**: 同一订单可能收到多次回调，需要幂等处理
2. **验签**: 富友返回的sign字段可能包含换行符，验签前需要 `.replace(/\s/g, '')`
3. **状态查询为主**: 回调为辅，最终状态以查询接口为准
