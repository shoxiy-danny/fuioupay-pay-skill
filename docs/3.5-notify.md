# 3.5 支付结果通知（回调）

## 请求地址

回调地址在请求报文的 `notify_url` 字段中指定，测试和生产环境都是如此。

## 场景说明

下单（主扫）交易的结果是以**异步**的形式进行回调的。富友在接收到支付宝等支付通道的回调结果以后再回调商户。

**重要**: 不保证通知最终一定能成功，在订单状态不明或者没有收到支付结果通知的情况下，建议商户主动调用【3.4 订单查询】确认订单状态。

**只有主扫统一下单、公众号/服务窗统一下单会通过此接口发异步通知，条码支付没有异步通知。**

## 回调格式

```
POST
Content-Type: application/x-www-form-urlencoded
req={encode之后的xml报文}
```

收到回调后直接在request里面获取`req`参数的值，然后decode一下就得到xml格式报文。

## 回调响应

商户接收回调成功处理成功后，直接往流里写入：**数字1**

**最多回调6次，每次间隔30秒。**

## 回调处理四步流程

收到异步回调后，必须按以下顺序处理，缺一不可：

1. **验签** — 用富友公钥验证回调真实性
2. **幂等判断** — 用 `hf_seq_id` 判断是否已处理过，避免重复操作
3. **二次查询** — 调用【3.4 订单查询】确认订单状态
4. **落库** — 确认后再更新本地订单状态

## 代码示例

```javascript
app.post('/callback', async (req, res) => {
  try {
    // ========== 第一步：解析并验签 ==========
    const encodedXml = req.body.req;
    const xmlData = doubleDecodeAndGbk(encodedXml);
    const data = await parseXml(xmlData);

    const cleanSign = data.sign.replace(/\s/g, '');
    const verifyContent = buildSignContent(data);
    const isValid = verifyWithPublicKey(verifyContent, cleanSign, publicKey);
    if (!isValid) {
      console.error('验签失败');
      return res.send('FAIL');  // 返回非1，富友会重试
    }

    // ========== 第二步：幂等判断 ==========
    const seqId = data.hf_seq_id;  // 富友返回的全局唯一流水号
    const exists = await db.query(
      'SELECT id FROM callback_log WHERE seq_id = $1',
      [seqId]
    );
    if (exists.rows.length > 0) {
      console.log('幂等：已处理过，直接返回成功');
      return res.send('1');
    }

    // ========== 第三步：二次查询确认 ==========
    const orderStatus = await queryOrderStatus(data.mchnt_order_no);
    if (orderStatus !== 'SUCCESS') {
      console.log('二次确认：订单未支付，忽略回调');
      return res.send('1');
    }

    // ========== 第四步：落库并记录 ==========
    await db.query(
      'INSERT INTO callback_log (seq_id, order_no, status) VALUES ($1, $2, $3)',
      [seqId, data.mchnt_order_no, 'SUCCESS']
    );
    await updateOrder(data.mchnt_order_no, 'success', data.order_amt);

    res.send('1');  // 返回1表示处理成功
  } catch (e) {
    console.error('回调处理异常:', e);
    res.send('FAIL');  // 富友会重试
  }
});
```

## 注意事项

1. **四步缺一不可**：验签→幂等→二次查询→落库，顺序不能乱
2. **幂等键**：用 `hf_seq_id`（富友全局唯一流水号）作为幂等判断依据
3. **二次确认**：回调成功不代表订单成功，必须主动查询确认
4. **验签sign含空格**：`.replace(/\s/g, '')` 不能漏
5. **返回非1会重试**：处理失败返回 `FAIL`，富友最多回调6次，间隔30秒
