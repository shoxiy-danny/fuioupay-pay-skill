# 3.13 微信委托代扣

## 请求地址

- 测试地址：`https://fundwx.payfuiouo2o.com/withhold`
- 生产地址：待富友提供

## 场景说明

委托代扣可应用于定期扣款或需事后扣款以期提高效率的场景。

例如离线刷脸快速支付，在用户授权给微信后，进行委托扣款的场景。

对于 trade_scene 取值为离线刷脸场景（K12_OFFLINE_FACE），此接口同步返回的为订单状态，机构也可通过代扣结果查询接口获取订单状态。

**本接口目前仅支持离线刷脸场景（K12_OFFLINE_FACE）**

| 07 | order_type | 必填 | String | 20 | 订单类型：WXPAP(微信委托代扣)目前仅支持此类型 |

| 13 | order_amt | 必填 | Number | 16 | 总金额，单位为分 |
| 14 | term_ip | 必填 | String | 128 | 终端IP |

### reserved_trade_scene 交易场景值

| 场景值 | 说明 |
|--------|------|
| PARKING | 车场停车场景 |
| PARKING_SPACE | 车位停车场景 |
| GAS | 加油场景 |
| HIGHWAY | 高速场景 |
| BRIDGE | 路桥场景 |
| LIFE_PAY | 生活缴费行业 |
| K12_OFFLINE_FACE | 离线刷脸代扣场景（目前仅支持此类型） |

### reserved_busi_data 字段说明

```json
{
    "scene_info": {...},
    "auth_appid": "wx8888888888888888",
    "contract_id": "Wx15463511252015071056489715",
    "auth_code": "离线刷脸扣款凭证"
}
```

| 序号 | 参数名 | 出现要求 | 型态 | 说明 |
|------|--------|----------|------|------|
| 01 | scene_info | 必填 | json | 场景信息值，不同业务场景设置不同的值 |

### reserved_trade_state 交易状态 (K12_OFFLINE_FACE场景)

| 状态值 | 说明 |
|--------|------|
| SUCCESS | 支付成功（无需处理） |
| REFUND | 转入退款（无需处理） |
| NOTPAY | 未支付（重试扣款） |
| CLOSED | 已关闭（换单号重试） |
| USERPAYING | 用户支付中（重试扣款） |
| PAYERROR | 支付失败（换单号重试） |

### reserved_debt_state 欠款状态

| 状态值 | 说明 |
|--------|------|
| NOT_DEBT | 无欠款（无垫资支付） |
| DEBT | 有欠款（垫资支付，有欠款） |
| REPAYMENT | 已还款（垫资支付，已还款） |
