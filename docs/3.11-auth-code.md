# 3.11 授权码查询openid

## 请求地址

| 环境 | 地址 |
|------|------|
| 测试 | `https://fundwx.payfuiouo2o.com/auth2Openid` |
| 生产 | `https://spay-cloud.fuioupay.com/auth2Openid` 或 `https://spay-cloud2.fuioupay.com/auth2Openid` |

## 场景说明

通过授权码换取用户的 openid（静默模式）。适用于微信公众号支付、微信小程序等场景，在用户授权后获取用户的唯一标识。

**典型应用场景**：
1. 微信 OAuth 静默授权后，获得授权码
2. 使用本接口将授权码兑换为用户的 openid
3. 将 openid 用于 JSAPI/小程序统一下单

## 请求报文

| 序号 | 参数名 | 必填 | 类型 | 长度 | 说明 |
|------|--------|------|------|------|------|
| 1 | version | 是 | String | 8 | 1.0 |
| 2 | ins_cd | 是 | String | 13 | 机构号 |
| 3 | mchnt_cd | 是 | String | 15 | 商户号 |
| 4 | term_id | 是 | String | 8 | 终端号 |
| 5 | random_str | 是 | String | 32 | 随机字符串 |
| 6 | sign | 是 | String | 512 | 签名 |
| 7 | auth_code | 是 | String | 128 | 授权码（微信授权code）|
| 8 | order_type | 是 | String | 20 | 订单类型：WECHAT/ALIPAY |
| 9 | term_ip | 是 | String | 40 | 终端IP |

## 响应报文

| 序号 | 参数名 | 必填 | 类型 | 说明 |
|------|--------|------|------|------|
| 1 | result_code | 是 | String | 错误代码，000000成功 |
| 2 | result_msg | 是 | String | 错误描述 |
| 3 | ins_cd | 是 | String | 机构号 |
| 4 | mchnt_cd | 是 | String | 商户号 |
| 5 | term_id | 否 | String | 终端号 |
| 6 | random_str | 是 | String | 随机字符串 |
| 7 | sign | 是 | String | 签名 |
| 8 | openid | 否 | String | 用户openid（成功时返回）|
| 9 | sub_openid | 否 | String | 子商户用户openid |
| 10 | buyer_id | 否 | String | 买家ID（支付宝场景）|

## 返回码

| result_code | 说明 | 处理方式 |
|-------------|------|----------|
| 000000 | 成功 | 获取到 openid，进行后续下单操作 |
| 000002 | 授权码无效 | 重新引导用户授权 |
| 其他 | 失败 | 查看 result_msg |

## 交易状态说明

此接口仅负责将授权码兑换为 openid，不涉及订单状态。

## 代码示例

```javascript
// 1. 先获取授权码（微信OAuth静默授权）
// 授权后回调URL中会携带 auth_code 参数

// 2. 使用授权码换取 openid
const result = await payClient.queryOpenIdByAuthCode({
  mchntCd: '000XXXXXXX',
  authCode: req.query.auth_code, // 微信授权code
  orderType: 'WECHAT',
  termIp: '127.0.0.1'
});

if (result.result_code === '000000') {
  console.log('openid:', result.openid);

  // 3. 使用 openid 进行 JSAPI 下单
  const orderResult = await payClient.jsapiPay({
    mchntCd: '000XXXXXXX',
    orderType: 'JSAPI',
    subOpenid: result.openid, // 传入获取到的 openid
    orderAmt: 1,
    goodsDes: '商品描述',
    notifyUrl: 'https://yourdomain.com/callback',
    termIp: '127.0.0.1'
  });
} else if (result.result_code === '000002') {
  // 授权码无效，需要重新授权
  console.error('授权码无效:', result.result_msg);
}
```

## 注意事项

1. **授权码有效期**：微信授权码有效期通常较短（几分钟），请尽快使用
2. **静默授权**：本接口适用于静默授权场景（scope=snsapi_base），用户无感知
3. **重复使用**：授权码一次性有效，使用后失效
4. **openid 缓存**：建议对 openid 进行缓存，避免重复换取
5. **订单类型**：微信渠道用 WECHAT，支付宝渠道用 ALIPAY

## 与 3.12 接口的区别

| 接口 | 说明 | 适用场景 |
|------|------|----------|
| 3.11 授权码查询openid | 通过授权码换取openid | OAuth授权后 |
| 3.12 服务商模式获取openid | 服务商模式下获取子商户用户openid | 服务商模式 |
