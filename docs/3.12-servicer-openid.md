# 3.12 服务商模式获取openid

## 请求地址

| 环境 | 地址 |
|------|------|
| 测试 | `https://fundwx.payfuiouo2o.com/oauth2/getOpenid` |
| 生产 | `https://spay-cloud.fuioupay.com/oauth2/getOpenid` 或 `https://spay-cloud2.fuioupay.com/oauth2/getOpenid` |

## 场景说明

在微信**服务商模式**下，通过授权码获取子商户用户的 openid。服务商模式与普通商户模式的区别在于：服务商需要通过子商户发起交易，资金清算到子商户账户。

**适用场景**：
1. 服务商模式下，代替子商户收单
2. 需要获取子商户用户的 openid 进行 JSAPI 支付
3. 子商户授权服务商代为发起交易

## 请求报文

| 序号 | 参数名 | 必填 | 类型 | 长度 | 说明 |
|------|--------|------|------|------|------|
| 1 | version | 是 | String | 8 | 1.0 |
| 2 | ins_cd | 是 | String | 13 | 机构号（服务商机构号）|
| 3 | mchnt_cd | 是 | String | 15 | 商户号（服务商商户号）|
| 4 | term_id | 是 | String | 8 | 终端号 |
| 5 | random_str | 是 | String | 32 | 随机字符串 |
| 6 | sign | 是 | String | 512 | 签名 |
| 7 | auth_code | 是 | String | 128 | 授权码（微信授权code）|
| 8 | order_type | 是 | String | 20 | 订单类型：WECHAT |
| 9 | term_ip | 是 | String | 40 | 终端IP |
| 10 | sub_mchnt_cd | 是 | String | 15 | 子商户号 |

## 响应报文

| 序号 | 参数名 | 必填 | 类型 | 说明 |
|------|--------|------|------|------|
| 1 | result_code | 是 | String | 错误代码，000000成功 |
| 2 | result_msg | 是 | String | 错误描述 |
| 3 | ins_cd | 是 | String | 机构号 |
| 4 | mchnt_cd | 是 | String | 服务商户号 |
| 5 | term_id | 否 | String | 终端号 |
| 6 | random_str | 是 | String | 随机字符串 |
| 7 | sign | 是 | String | 签名 |
| 8 | openid | 否 | String | 用户openid（成功时返回）|
| 9 | sub_openid | 否 | String | 子商户用户openid（服务商模式下使用此字段）|
| 10 | buyer_id | 否 | String | 买家ID |

## 返回码

| result_code | 说明 | 处理方式 |
|-------------|------|----------|
| 000000 | 成功 | 获取到子商户用户openid，进行后续下单操作 |
| 000002 | 授权码无效 | 重新引导用户授权 |
| 其他 | 失败 | 查看 result_msg |

## 代码示例

```javascript
// 服务商模式：通过授权码获取子商户用户openid
const result = await payClient.servicerQueryOpenId({
  mchntCd: '服务商商户号',
  subMchntCd: '子商户号',      // 必填：子商户号
  authCode: req.query.auth_code, // 微信授权code
  orderType: 'WECHAT',
  termIp: '127.0.0.1'
});

if (result.result_code === '000000') {
  console.log('子商户用户openid:', result.sub_openid);

  // 使用 sub_openid 进行服务商模式JSAPI下单
  const orderResult = await payClient.jsapiPay({
    mchntCd: '服务商商户号',
    subMchntCd: '子商户号',    // 子商户号
    subOpenid: result.sub_openid, // 子商户用户openid
    orderAmt: 1,
    goodsDes: '服务商代子商户收款',
    notifyUrl: 'https://yourdomain.com/callback',
    termIp: '127.0.0.1'
  });
}
```

## 服务商模式说明

### 角色说明

| 角色 | 说明 |
|------|------|
| 服务商 | 帮助子商户发起交易的平台方 |
| 子商户 | 实际提供商品/服务的商户 |

### 交易流程

```
1. 用户在子商户页面选择微信支付
2. 服务商代替子商户调用统一下单
3. 通过本接口获取子商户用户的 sub_openid
4. 使用 sub_openid 调起微信支付
5. 资金清算到子商户账户
```

### 与普通模式区别

| 字段 | 普通模式 | 服务商模式 |
|------|----------|------------|
| mchnt_cd | 商户自己的商户号 | 服务商商户号 |
| sub_mchnt_cd | 不填 | 必填子商户号 |
| openid | 用户在该商户的openid | 不使用 |
| sub_openid | 不填 | 必填：子商户用户openid |

## 注意事项

1. **子商户必须先授权**：子商户需要在服务商平台完成授权，才能代为发起交易
2. **sub_openid 必须使用**：服务商模式下下单必须使用 `sub_openid`，不能使用普通 `openid`
3. **资金清算**：交易资金直接清算到子商户账户，服务商收取服务费
4. **授权码有效期**：与普通模式一致，授权码有效期较短，请尽快使用
5. **order_type 仅支持 WECHAT**：目前服务商模式仅支持微信渠道

## 与 3.11 接口的区别

| 接口 | 说明 | 适用场景 |
|------|------|----------|
| 3.11 授权码查询openid | 普通商户获取用户openid | 普通商户模式 |
| 3.12 服务商模式获取openid | 服务商获取子商户用户openid | 服务商模式，需填 sub_mchnt_cd |
